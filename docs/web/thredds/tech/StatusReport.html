THREDDS Technical Status

THREDDS and Unidata

The efforts of the THREDDS group focus mainly on three themes from the Unidata 2013 proposal:

  • Theme 2 - Advancing data services;
  • Theme 3 - Developing and deploying useful tools; and
  • Theme 5 - Providing leadership in cyberinfrastructure.

The work of the THREDDS group is comprised of two main areas: the THREDDS Data Server (TDS) and the Common Data Model (CDM) / netCDF-Java library. The data available from the IDD is a driving force on both the TDS and netCDF-Java development. The ability to read all the IDD data through the netCDF-Java library allows the TDS to serve that data and provide services on/for that data. There is also a strong tie with the IDV group because they depend to a large degree on the netCDF-java library for access to the data.

Status update 7 April 2011

THREDDS Data Server (TDS)

  • TDS currently at 4.2.5. Some major features since 4.2 first released include:
    • Distribution now includes the ncISO metadata services (thanks to Dave Neufeld and Ted Habermann, NOAA/NGDC)
    • Add OPeNDAP handling of ESG access control (other access control improvements)
    • Subsetting services for Point Obs data extended to Station data (other improvements as well)
  • TDS 4.2 became our stable release on 22 November 2010. The main new features were:
    • Refactor FMRC aggregation to simplify configuration and improve performance.
    • Improved integration with ncWMS code including adding the Godiva2 browser based WMS client.
    • Addition of experimental subsetting services for Point Obs data (Discrete Sampling data)

CDM / netCDF-Java library

  • CDM 4.2.23 is our current release. The main features since 4.2.0 include:
    • The Discrete Observations Convention is now complete, should be added to CF-1.6.
    • Nexrad2: dual pole radar
    • GradsBinaryGridServiceProvider (Don Murray)
    • Netcdf-Java can now be built with maven. It will be added to the standard maven repository.
    • Opendap parser now uses the same grammer as the C library.
  • CDM 4.2 is now our stable release and includes:
    • FMRC aggregations have been refactored to dynamically adjust to changes in model output, eg from the NCEP IDD feed. The FMRC "Best" dataset can be parameterized by forecast time. The TDS admin can choose which FMRC datasets to expose in the catalogs.
    • A new abstraction was developed for managing large collections of data, which can be used by NcML and other packages.
    • Caching of Grid dataset info.
    • Caching of File System info.
    • GRIB processing now correctly identifies time interval variables such as accumulations, averages, min/max over intervals, etc. This feature was propagated up the CDM stack, eg into FMRC aggregations.
    • OPeNDAP parsing is now done from the same grammar as with the OPeNDAP/netCDF C library.
    • ncstream works for the full netcdf-4 data model. performance tests are very encouraging.
  • Other CDM related activities
    • present at 4th Extremely Large Databases Conference (XLDB)
    • present at USGS Community for Data Integration Workshop

Status update 1 March 2010

TDS 4.1 and CDM 4.1 are the current stable releases. The features and enhancements that have gone into those versions as well as plans for both 4.2 versions are given below.

  • THREDDS Data Server (TDS)
    • TDS 4.1 was promoted to our stable release in December. Includes:
      • Improved tracking of dataset collections (used in aggregation and PointObs collections) which affects performance on large data collections.
      • CdmRemote - experimental web service for point feature datasets
      • ncstream - experimental web service for remote access to CDM datasets
      • Improved handling of GRIB ensemble data (including a new GRIB index format).
      • Performance, memory, and stability improvements
      • Uses netCDF-Java/CDM 4.1
    • Plans for TDS 4.2 development
      • Refactor FMRC aggregation to simplify configuration and improve performance.
      • Improved integration with ncWMS code (easier to stay up-to-date with ncWMS code).
      • Further development of Point Feature collection services including extending it to other data types.
      • Continued refactoring of TDS configuration system for ease of configuration and improved performance.
  • Common Data Model / netCDF-Java
    1. Netcdf-Java library version 4.1 is now the stable release
      1. Point Features
        1. CF-1.5 (provisional), Suomi, FslWindProfiler, Cosmic, GempakSurface, Madis, UnidataObs, GempakSounding, NLDN, USPLN, Madis-ACARS
        2. Handle multiple profiles in the FslRaob plugin
        3. Add new functionality to StationProfileFeature: getTimes(), getProfileByDate()
      2. Coordinate Systems
        1. CEDRIC radar
        2. Orthogonal Curvilinear (GRIB2)
        3. MSGnavigation (GRIB2 from EUMETSAT)
        4. AlbersEqualAreaEllipse, LambertConformalConicEllipse
      3. IOSP
        1. H5iosp: deal with compressed, chunked compound types
        2. BUFR: use WMO (prerelease) tables version 14, many other improvements
        3. GEMPAK sounding IOSP
        4. Improve HDF-EOS processing
      4. ToolsUI
        1. New Tabs for for examining BUFR tables B and D
      5. General
        1. Add -isLargeFile option to FileWriter to generate 64-bit offset files
        2. Experimental filesystem caching
    2. Netcdf-Java library version 4.2 is the experimental release
      1. FMRC refactor
        1. No longer have to specify file structure, all files are scanned as GridDatasets and FMRC is automatically created
        2. no longer need fmrcDefinition.xml files, equivilent is generated automatically
        3. UI Tabs for examining FMRC data structures
        4. experimental caching using Berkely DB Java edition
      2. Collections specification
        1. same functionality as scan, scanFmrc, and CrawlableDataset
        2. specify a collection with a single string, eg " /data/NCEP/GFS/Alaska_191km/**/GFS_Alaska_191km_#yyyyMMdd_HHmm#\.grib"
        3. specify collection from a THREDDS catalog
        4. specify collection from NcML
      3. Improve support for arbitrary 2D Coordinates, like Orthogonal Curvilinear
    3. Six month plan:
      1. Release version 4.2 with refactored FMRC
      2. Complete CF point obs conventions, for CF version 1.5
      3. Complete implementation for CF point obs.
      4. Complete point collection services infrastructure
      5. (Possible) JNA interface to netcdf-4 C library for writing netcdf-4.

Status update 07/02/2007

  1. THREDDS Data Server (TDS): Current release is 3.16.
    1. The NetCDF Subset Service is an experimental REST web service for subsetting CDM scientific datasets. The subsetting is specified using earth coordinates, such as lat/lon bounding boxes and date ranges, rather than index ranges that refer to the underlying data arrays. The data arrays are subsetted but not resampled or reprojected, and preserve the resolution and accuracy of the original dataset.
    2. New authorization/authentication capabilities allow servers to be configured with restricted dataset access.
      • Can use CAS, CAMS or other pluggable authorization
    3. Forecast Model Run Collections : Automatically create 1D subsets of 2D time datasets. (Example)
    4. Runtime configuration completed, allows users to configure, load classes at runtime.
    5. Dataset Source plugin: allows users to create custom and virtual datasets.
    6. Refactor of dataset filter interface (CrawlableDatasetFilter) simplifies new implementations
    7. TDS Tutorial will be updated for next Unidata workshop.
    8. OPeNDAP server: direct implementation of ascii ascii response (not recursive call)

  2. Common Data Model: Current release is 2.2.20, last stable release using Java 1.4.
