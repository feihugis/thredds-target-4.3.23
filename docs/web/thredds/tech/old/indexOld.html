Data Provider Tools

THREDDS Data Server

THREDDS Internet Data Distribution (IDD) Server

Much of the realtime data available over the Unidata Internet Data Distribution (IDD) is available through a THREDDS Data Server hosted at Unidata on motherlode.ucar.edu. You are welcome to browse and access these meteorological datasets.

If you have an IDD running, you can run your own THREDDS/IDD Server. We have written standard THREDDS/IDD pqact files that have corresponding catalogs. You should download and install the latest TDS version, and follow these instructions.

THREDDS Catalog Generator

The THREDDS Catalog Generator can produce a THREDDS catalog by scanning or crawling one or more local or remote dataset collections. The information it needs to know about the dataset collection and the resulting catalog are provided by a configuration document.

The Catalog Generator can be run as a Java Servlet or as a stand-alone Java application. The application reads a single configuration document and produce a THREDDS catalog. The servlet provides 1) scheduling of all the catalog generation tasks being performed by the servlet, and 2) a web interface to control the configuration of the CatalogGen servlet.

See the THREDDS Catalog Generator page for more information.

Catalog Services :

  2. You can view any Dataset Inventory Catalog as an HTML document with the Show catalog as HTML form.

Writing Dataset Query Capability (DQC) Services

THREDDS DQC Services provide access to DQC documents and respond to the queries allowed by the DQC documents. See the THREDDS DQC Servlet page for more information.

THREDDS XML

Dataset Inventory Catalog

THREDDS Catalogs are logical directories of on-line data resources, encoded as XML documents, which provide a place for annotations and other metadata about the data resources to reside. This is how THREDDS-enabled data consumers find out what data is available from data providers. These catalogs can be hand-generated and simply placed on a web server for distributed access. They can also be dynamically generated and serve as a front end to large data portals. Because they are XML rather than HTML, they are much easier to read by application programs.

The Catalog 1.0.1 specification is an annotated XML Schema that explains the details of THREDDS Catalogs. Online catalog 1.0.1 schema .

The latest version is 1.0.1 is used by the THREDDS Data Server and is described in Catalog 1.0.1 changes.

Dataset Query Capabilities (DQC) XML

This is an experimental XML document format that describes the possible ways an end-user can choose dataset subsets. These choices are formed into a query to a DQC Server that returns a catalog of actual dataset URLs that satisfy that query. See the DQC Specification for details.

Some example DQCs:

  • Radar Images (station, product, time)
  • Radar Images using macros
  • ADDE Images (group/dataset, time)

THREDDS Clients

Integrated Data Viewer:

Unidata's IDV is a 3D visualization application based on VisAD that is our flagship client for THREDDS data services.

"The IDV brings together the ability to display and work with satellite imagery, gridded data (for example, numerical weather prediction model output, and many Climate Diagnostics Center grids), surface observations (METARs), balloon soundings, NWS NEXRAD Level II and Level III RADAR data, and NOAA National Profiler Network data, all within a unified interface. The IDV can display any Earth-located data if it is provided in a known format "

NetCDF UI Tools:

This is a simple "data browsing" application for testing and debugging NetCDF files and THREDDS catalogs and data services. It is also the testing ground for various client side UI widgets, tools, cranks and bric-a-brac that we are developing at Unidata. You can download it or launch it via WebStart.

THREDDS Client Library

The Netcdf-Java 2.2 library has all the code needed to parse catalogs etc. Use it in your client programs to access THREDDS catalogs.
