Cook Book for THREDDS installation

THREDDS can be deployed with different configurations depending on the dataset and the server(s) used to distribute the data. The dataset could be as large as the NOAAport feed, a research result or any other type of data. The server could be a known type such as OPeNDAP (DODS) or a proprietary server specialized for the particular dataset.

All deployments need to create THREDDS catalogs for the datasets so the datasets can be published , discovered and accessed in a convienent manner.

Data Server with THREDDS Catalogs

This deployment requires that the THREDDS catalogs are create by the site manually or a script designed by the site. Also the site is required to supply the Data Server for the dataset that understands how to interpet/use THREDDS catalogs. All configurations are the responsibility of the site.

Data Server with THREDDS Cataloging Package

Same as above except the THREDDS catalogs are generated by the Catalog Generator. The Catalog generator needs to be configured to crawl the dataset(s). The Catalog Generator adds the flexiblility to create catalogs on non-static datasets without human intervention, keeps the catalogs insync with the dataset(s).

THREDDS-IDD Server Package

This deployment supplies all the features currently on Unidata's THREDDS server that distributes the NOAAport data and casestudy data using a variety of data formats and multiple data servers. This installation requires extensive infrastructure but after install the site will have the tools available to add other datasets and to manipulate data.
