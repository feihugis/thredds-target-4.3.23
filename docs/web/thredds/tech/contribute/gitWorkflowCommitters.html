Git Workflow: Authorized Committers

If you are authorized to commit to the Unidata/thredds repository, ...

Overview of Workflow

Now that you have identified an issue or feature to work on, these are the steps we follow to commit changes to the Unidata thredds repository on gitHub.

  1. Make sure your local repository is up to date with Unidata/thredds repository (origin):
    • If you are new to working on the THREDDS code:
      • Clone the Unidata/thredds repository to create a local copy of the repository.
      • Check out the current development branch ("target-4.3.13" as of 6 Aug 2012)
    • If you already have a fork and a local repository:
      • Fetch any changes from origin.
      • If there are changes from origin/target-4.3.13:
        • Merge those commits onto local/target-4.3.13.
  2. Create a local branch (e.g., local/myCoolFix) off of the current development branch (origin/target-4.3.13)
  3. Make changes to code, commit changes frequently to your local branch ("myCoolFix").
  4. Make sure your branch is up to date with origin/target-4.3.13.
    • Fetch any commits from origin/target-4.3.13.
    • If there are changes from origin/target-4.3.13, merge those commits into local/target-4.3.13.
    • Rebase local/myCoolFix onto, the now up-to-date, local/target-4.3.13. [NOTE: Use interactive rebase to clean up commit history, as desired.]
  5. Repeat the last two steps as needed.
  6. Once you have tested your changes locally and want to share your changes with the rest of the development team:
    • Make sure your branch is up to date with origin/target-4.3.13 ...
    • Push your changes from local/myCoolFix to origin/target-4.3.13
  7. Update your local development branch:
    • Checkout the development branch, local/target-4.3.13
    • Pull from origin/target-4.3.13
  8. Remove local/myCoolFix branch
