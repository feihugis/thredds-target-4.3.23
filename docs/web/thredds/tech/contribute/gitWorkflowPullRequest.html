Git Workflow: Making Pull Requests

GitHub has very nice support for requesting your changes be pulled to an origin server.

Overview of Workflow

Now that you have identified an issue or feature to work on, these are the steps we follow to make a pull request to the Unidata thredds repository on gitHub.

  • Make sure your gitHub fork and your local repository are up to date with Unidata/thredds repository:
    • If you are new to working on the THREDDS code:
      • Create your own gitHub fork of the Unidata/thredds repository, myUserName/thredds.
      • Clone your fork to create a local copy of the repository.
      • Add the Unidata/thredds repository as a remote. Ending up with:
        • remote: "origin" == git@github.com:myUserName/thredds.git (read+write)
        • remote: "unidata" == git://github.com/Unidata/thredds.git (read only)
      • Check out the current development branch ("target-4.3.12" as of 6 Aug 2012)
    • If you already have a fork and a local repository:
      • Fetch any commits from Unidata/target-4.3.12.
      • If there are changes from Unidata/target-4.3.12:
        • Merge those commits into the local development branch.
        • Push those commits to your fork, origin/target-4.3.12.
  • Create a local branch (e.g., "myCoolFix") off of the current development branch
  • Commit changes to your local branch ("myCoolFix"), repeat as needed.
  • Once you've tested your changes locally and want to share your changes with the rest of the development team, make sure your copy of the development branch is up to date.
    • Fetch any commits from Unidata/target-4.3.12.
    • If there are changes from Unidata/target-4.3.12:
      • Merge those commits into the local development branch, local/target-4.3.12.
      • Push those commits to your fork, origin/target-4.3.12.
  • Rebase your local branch ("myCoolFix") onto the HEAD of the development branch.
  • Push your changes from your local branch, local/myCoolFix, to that branch on your fork, origin/myCoolFix
  • Make a pull request from origin/myCoolFix to unidata/target-4.3.12
  • ... any suggested changes, etc.
  • Once your change is accepted:
    • Update your local development branch
    • Remove local/myCoolFix branch
    • Remove origin/myCoolFix branch
  • Create your own gitHub fork of the Unidata thredds repository
  • Clone your fork locally
  • Create a local branch (myCoolFix) off of target-4.3.12
  • work and commit to myCoolFix
  • Fetch any commits from origin/target-4.3.12
  • Rebase -i
  • push to origin/target-4.3.12
  • pull from origin/target-4.3.12
  • Remove local/myCoolFix branch
