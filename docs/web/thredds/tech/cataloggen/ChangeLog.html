ChangeLog - THREDDS Catalog Generator

Changes from 0.9 to 0.9.3

  • Remove all empty collection datasets before returning the catalog from DatasetSource.expand().
  • Improve how a CatGen config document and the generated datasets are merged, mainly involved the dropping of the top-level generated dataset.
  • Allow dataset filters to filter out as well as accept datasets.
  • Add DirectoryScanner for simple API to catGen.
  • Add catalogRef capabilities, currently either expand all or create catalogRef for all subdirectories.
  • Refactor DatasetSource abstract methods: replace expandThisType() with createDataset(), isCollection(), createSkeletonCatalog(), and expandThisLevel().

Changes from 0.8.2 to 0.9

  • Simplify CatGenConfig 0.5 DTD and XSD. Now more closely reflects what CatalogGen software supports.
  • Make dataset filtering more flexible and work better on collection datasets. Includes adding new datasetFilter attributes to CatGenConfig 0.5 DTD and XSD (matchPatternTarget,applyToCollectionDataset, applyToAtomicDataset, invertMatchMeaning).

Changes from 0.8.1 to 0.8.2

  • Clean up when servlet is destroyed (e.g., when Tomcat is shutdown)

Changes from 0.8 to 0.8.1

  • Update for new thredds.catalog object model (now handles InvCatalog 1.0 and 0.6).

Changes from 0.7.5 to 0.8

  • Update for changes to the thredds.catalog object model (still at InvCatalog 0.6).
  • Update thredds.servlet.ServletUtil logging to use straight log4j rather than thredds.util.Log and add a number of context level parameters to configure logging.
  • Converted CatGenServlet to the THREDDS servlet framework
    • Use thredds.servlet.ServletUtil
    • Moved editable content (config, etc) into initial directory
    • Added to web.xml for other servlets
  • Updated example servlet config files
    • added a DODS datasource example
    • added a test data directory to distribution and an example that uses it as a datasource
  • Modified thredds/cataloggen/build.xml:
    • clean up several parameters
    • modify the construction of the war file directory
    • add several targets to run individual tests.
    • include documentation files in war files (also served by servlet)
  • Modified thredds/server/build.xml:
    • introduce the compile.libraries fileset to clean up some redundancy;
    • call the cataloggen build.xml file to construct the war file
  • Updated web.xml (cataloggen and server versions) to include all THREDDS servlets, moved the logging parameters from the servlet to the context level, added security constraints for cataloggen POST actions.
