Implementing CrawlableDataset and Related Interfaces



CrawlableDataset

If your dataset collection is not a collection of data files, you will have to implement CrawlableDataset for your collection before you can use the TDS catalog generation framework. The CrawlableDatset interface is fairly straightforward so this shouldn't be too difficult.

Though not part of the interface, for your implementation of CrawlableDataset to work in the "TDS catalog generation framework", a constructor with two parameters (one a String, the other an Object) is required. The String parameter is the CrawlableDataset path. The Object parameter is a configuration object (a single org.jdom.Element) that your implementation can use as it wishes. This is documented in the javadoc documentation.

To use your implementation in the "TDS catalog generation framework", simply include a crawlableDatasetImpl element in the appropriate datasetScan with a className attribute that gives the name of your class. A single child element of the crawlableDatasetImpl element will be passed to the String/Object constructor mentioned above as an org.jdom.Element.

CrawlableDatasetFilter

A CrawlableDatasetFilter implementation can be used in the "TDS catalog generation framework" in a way similar to how a CrawlableDataset implementation can be used. In this case, the constructor that is called has a single configuration Object parameter, again handed a single org.jdom.Element.

To use your implementation in the "TDS catalog generation framework", simply include a crawlableDatasetFilterImpl element in the filter element with a className attribute that gives the name of your class. A single child element of the crawlableDatasetFilterImpl element will be passed to the constructor mentioned above as an org.jdom.Element.
