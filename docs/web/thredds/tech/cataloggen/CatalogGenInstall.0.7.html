THREDDS Catalog Generator - Version 0.7

Comments to Ethan Davis or THREDDS mail list

Version 0.7.5
This software is currently an alpha release (as of 24 September 2003).

The THREDDS Catalog Generator is a Java application that, given a configuration file (see below), produces a THREDDS catalog (or Aggregation Server Config file). The configuration file specifies the details needed to scan or crawl existing dataset sources. Currently, only local directories are crawled; future versions will crawl DODS servers, DODS file servers, etc.

Download

  • Required Libraries
    • Catalog Generator 0.7 [Download cataloggen-0.7.5.jar]
    • THREDDS library [included in the cataloggen.jar file]
    • DODS Java 1.1.1 [Download dods.1.1.1.jar]
    • JDOM version beta8 [Download jdom-b8.jar]
    • Log4j version 1.2.7 [Download log4j-1.2.7.jar]
  • Source, cataloggen-0.7.5.src.jar
    • which, to run the test suite, requires JUnit [Download junit-3.8.1.jar]

Setup

Download the libraries listed above. To run the catalog generator, see the "Usage" section below; just make sure that the libraries listed above are all on your CLASSPATH. For instance, I use the java option "-classpath cataloggen-0.7.5.jar:jdom-b8.jar:dods.1.1.1.jar:log4j-1.2.7.jar".

Usage

java thredds.cataloggen.CatalogGen [options] <configDocName> [<outputFileName>]

Given a CatalogGenConfig 0.5 document that describes a dataset source to be scanned or crawled, produce an InvCatalog 0.6 document that lists the datasets from the specified dataset source. The resulting InvCatalog document will be sent to standard output unless an output file is specified. Note: the configuration document may be specified as a local file name or as a URL.

Options:

  • -help
    Print a usage message.
  • -log <logFileName> <logLevel>
    Write a log file at the given log level (OFF, FATAL, WARN, INFO, DEBUG, ALL).

Configuration File

You will need to write a CatalogGen configuration document that describes how to access the datasets you want to catalog, how to group those datasets, and how to name those datasets. The syntax and meaning of the CatalogGen config files are described on the "CatalogGen Configuration" page.
