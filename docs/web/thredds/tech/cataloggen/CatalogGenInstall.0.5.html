THREDDS Catalog Generator - Version 0.5

Comments to Ethan Davis or THREDDS mail list

Version 0.5.1 (using THREDDS InvCatalog 0.6)
This software is currently an alpha release (as of 20 August 2002).

The THREDDS Catalog Generator is a Java application that, given a configuration file (see below), produces a THREDDS catalog (or Aggregation Server Config file). The configuration file specifies the details needed to scan or crawl existing dataset sources. Currently, only local directories are crawled; future versions will crawl DODS servers, DODS file servers, etc.

Download

  • Required Libraries
    • Catalog Generator 0.5.1 [Download cataloggen-0.5.1.jar]
    • An XSLT processor that supports certain XSLT 1.1 features. Currently, the only one I am aware of is the Saxon XSLT processor. I am using version 6.5.2 (which I'm making available here or you can get from the Saxon page listed above). [NOTE: The feature required is to treat temporary trees as node sets instead of temporary tree fragments. The problem is that temporary trees can't be used in most places that node sets can be used (basically you can only output a temporary tree to the output document). This makes it difficult to perform more complex transformations.]
    • DODS Java 1.1.1 [Download dods.1.1.1.jar] [NOTE: Currently only using GNU RegExp from the DODS library but will be adding some DODS functionality, e.g., crawling DODS servers and DODS file server, in the future.]
  • Source, cataloggen-0.5.1.src.jar

Usage

  • java thredds.cataloggen.CatalogGen <configFileName>
    Given a CatalogGenConfig 0.5 document that specifies the details needed to scan or crawl existing dataset sources, produce a InvCatalog 0.6 document with the details of the given dataset sources.
  • java thredds.cataloggen.CatalogGen -gdsCat <gdsXmlUrl>
    Transforms the given GDS XML catalog into a THREDDS catalog, InvCatalog 0.6.
  • java thredds.cataloggen.CatalogGen -expandAny <configFileName>
    Same as with no option except the configuration document is pre-CatalogGenConfig0.5 (see CatalogGen 0.2), the result is a InvCatalog 0.6 document.
  • java thredds.cataloggen.CatalogGen -expandAny5 <configFileName>
    Same as the '-expandAny' option except the result is a a InvCatalog 0.5 document.
  • java thredds.cataloggen.CatalogGen -help
    Print a usage message.

Note: Be sure that cataloggen-0.5.jar, a XSLT processor class library, and dods.1.1.jar are all on your CLASSPATH. For instance, I use the java option "-classpath cataloggen-0.5.jar:saxon-6.5.2.jar:dods.1.1.1.jar".

The Configuration File

A CatalogGen configuration document is a THREDDS catalog used to specify details needed to scan or crawl existing dataset sources and produce a THREDDS catalog with the details of the given dataset sources.

See the "CatalogGen Configuration" page for more details on the configuration files.
