THREDDS Catalog Generator - Version 0.6

Comments to Ethan Davis or THREDDS mail list

Version 0.6
This software is currently an alpha release (as of 13 November 2002).

The THREDDS Catalog Generator is a Java application that, given a configuration file (see below), produces a THREDDS catalog (or Aggregation Server Config file). The configuration file specifies the details needed to scan or crawl existing dataset sources. Currently, only local directories are crawled; future versions will crawl DODS servers, DODS file servers, etc.

Download

  • Required Libraries
    • Catalog Generator 0.6 [Download cataloggen-0.6.jar]
    • THREDDS library [Download thredds.jar]
    • DODS Java 1.1.1 [Download dods.1.1.1.jar]
    • JDOM version beta8 [Download jdom-b8.jar]
  • Source, cataloggen-0.6.src.jar

Setup

Download the libraries listed above. To run the catalog generator, see the "Usage" section below; just make sure that the libraries listed above are all on your CLASSPATH. For instance, I use the java option "-classpath cataloggen-0.6.jar:thredds.jar:jdom-b8.jar:dods.1.1.1.jar".

Usage

  • java thredds.cataloggen.CatalogGen <configDocURL> [<outputFile>]
    Given a URL for a CatalogGenConfig 0.5 document, that specifies the details needed to scan or crawl existing dataset sources, produce a InvCatalog 0.6 document with the details of the given dataset sources. The resulting InvCatalog document will be sent to standard output unless an output file is specified. NOTE: the <configDocURL> can be a local file URL ("file:").
  • java thredds.cataloggen.CatalogGen -help
    Print a usage message.

Configuration File

You will need to write a CatalogGen configuration document that describes how to access the datasets you want to catalog, how to group those datasets, and how to name those datasets. The syntax and meaning of the CatalogGen config files are described on the "CatalogGen Configuration" page.
